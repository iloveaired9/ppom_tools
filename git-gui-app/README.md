# Git GUI - 웹 기반 Git 관리 도구

간단하고 직관적인 웹 인터페이스에서 Git 저장소를 관리하세요. 터미널 명령어 없이 브라우저만으로 커밋 히스토리, 브랜치 관리, 파일 변경 추적이 가능합니다.

## 주요 기능

✨ **Git 상태 대시보드**
- 현재 브랜치 확인
- 수정된/추적되지 않은/스테이징된 파일 개수 표시
- 원격 저장소와의 커밋 차이 표시

📝 **커밋 히스토리**
- 모든 커밋 목록 조회
- 작성자, 날짜, 메시지 표시
- 최대 50개 커밋 한 번에 로드

🌿 **브랜치 관리**
- 모든 브랜치 목록 조회
- 현재 브랜치 표시
- 브랜치 전환 기능

📁 **파일 관리**
- 특정 파일을 이전 버전으로 복원
- 변경사항 손실 확인

## 설치 및 실행

### 요구사항
- Node.js 12.0 이상
- Git이 설치되어 있어야 함
- Windows, macOS, Linux 모두 지원

### 설치

1. **의존성 설치**
```bash
npm install
```

2. **서버 시작**
```bash
npm start
```

3. **브라우저 접속**
```
http://localhost:3000
```

## 사용 방법

### 1단계: 저장소 등록
- "저장소 설정" 섹션에 Git 저장소 경로 입력
- 예시:
  - Windows: `C:\Users\username\project` 또는 `C:\Users\username\Documents\myrepo`
  - macOS/Linux: `/Users/username/project` 또는 `/home/user/project`
- "저장소 로드" 버튼 클릭

### 2단계: 저장소 상태 확인
- "상태" 탭에서 현재 브랜치와 파일 변경 상황 확인
- 자동으로 5초마다 새로고침됨

### 3단계: 커밋 히스토리 조회
- "커밋" 탭에서 모든 커밋 확인
- 커밋 클릭으로 메시지 상세 보기

### 4단계: 브랜치 관리
- "브랜치" 탭에서 모든 브랜치 확인
- "전환" 버튼으로 브랜치 변경

### 5단계: 파일 복원
- "파일" 탭에서 복원할 파일명 입력
- "파일 복원" 버튼 클릭
- 확인 메시지 후 이전 버전으로 복원

## API 엔드포인트

### 설정
- `POST /api/configure` - 저장소 경로 설정
- `GET /api/config` - 현재 설정 조회

### 상태 및 정보
- `GET /api/status` - 저장소 상태
- `GET /api/commits` - 커밋 히스토리
- `GET /api/branches` - 브랜치 목록

### 작업
- `POST /api/switch-branch` - 브랜치 전환
- `POST /api/revert` - 파일 복원
- `GET /api/diff/:file/:commit1/:commit2` - 파일 비교

## 설정

### 저장소 경로 변경
- 앱을 재실행하면 마지막 설정한 경로가 자동으로 로드됩니다.
- 새로운 경로를 설정하려면 "저장소 설정" 섹션에서 다시 입력하세요.

### 포트 변경
`server.js` 파일에서 `PORT` 값을 수정하세요:
```javascript
const PORT = 3000; // 원하는 포트로 변경
```

## 보안 주의사항

⚠️ **중요**: 이 도구는 로컬 개발용입니다.

- **로컬 네트워크만 사용**: http://localhost:3000에서만 실행하세요
- **원격 접속 금지**: 인터넷에 노출되지 않도록 주의하세요
- **인증 없음**: 접근 제어가 없으므로 신뢰할 수 있는 환경에서만 사용하세요
- **파일 변경 주의**: 복원 작업은 되돌릴 수 없으므로 신중하게 사용하세요

## 지원되는 Git 작업

✅ 읽기 작업 (완전 지원)
- 저장소 상태 확인
- 커밋 히스토리 조회
- 브랜치 목록 확인
- 파일 변경 사항 확인

✅ 쓰기 작업 (제한적 지원)
- 브랜치 전환
- 파일 복원

❌ 미지원 작업
- Commit 생성
- Stash 관리
- Tag 관리
- Merge/Rebase
- Push/Pull

이 기능들이 필요하면 터미널에서 Git 명령어를 사용하세요.

## 문제 해결

### "저장소가 설정되지 않았습니다" 에러
→ 저장소 경로를 정확히 입력하고 "저장소 로드"를 클릭하세요.

### "Git 저장소가 아닙니다" 에러
→ 입력한 경로가 올바른 Git 저장소인지 확인하세요. 
  `.git` 폴더가 있어야 합니다.

### 저장소 경로를 찾을 수 없음
→ 경로에 공백이나 특수문자가 있으면 전체 경로를 큰따옴표로 감싸세요.

### 포트 3000이 이미 사용 중
→ `server.js`에서 다른 포트로 변경하세요.

## 기술 스택

- **백엔드**: Node.js + Express.js
- **프론트엔드**: HTML5 + CSS3 + Vanilla JavaScript
- **Git 통신**: Node.js child_process
- **응답 형식**: JSON
